**五、项目经验------离线优先同步机制**

**5.1 项目背景**

候选人在项目中自研了一套基于 WorkManager
的离线优先数据同步机制，核心目标是：

-   保证增量数据不出现差错（版本号与后端对齐）

-   感知后端新数据并及时同步到本地

-   对实时性要求高的操作（点赞、收藏）区别处理

**5.2 架构设计要点**

  --------------------- -------------------------------------------------------------------
  **同步器设计**        继承 CoroutineWorker，在 doWork 中执行同步逻辑；通过扩展函数 +
                        自定义 Syncable 接口，确保同步方法只能在同步器内调用，防止滥用。

  **版本控制**          在 DataStore 中存储
                        lastVersion，同步成功后更新，失败则不更新，保证与后端版本号一致。

  **分级同步策略**      点赞/收藏：先写本地 DB 标记未同步，启动周期性 Worker
                        轮询上传，优先保证实时性。帖子创建：有网时直接发起请求（expedited
                        task），无网则入队等待。拉取同步：每次用户打开 App
                        时，将后端最新数据拉取至本地，保证单一数据源。
  --------------------- -------------------------------------------------------------------

**六、算法题：带过期时间的 LRU 缓存**

**6.1 题目描述**

实现一个支持过期时间的 LRU 缓存，容量为 N。规则如下：

-   缓存超过 N 个 KV 时，若存在过期的 KV，则随机淘汰一个过期项；

-   若无过期项，则按 LRU 规则淘汰最久未使用的 KV；

-   每个 KV 可设置独立的过期时间（TTL）。

**6.2 候选人思路**

候选人选择使用双向链表 + HashMap 实现 LRU
的核心结构，并在每个节点中存储过期时间戳，以支持 TTL
判断。整体思路清晰，符合 LRU 经典实现方案。

**6.3 参考实现（Kotlin）**

data class Node\<K, V\>( val key: K, var value: V, var expireAt: Long =
Long.MAX_VALUE, var prev: Node\<K, V\>? = null, var next: Node\<K, V\>?
= null ) class LRUCache\<K, V\>(private val capacity: Int) { private val
map = HashMap\<K, Node\<K, V\>\>() private val head = Node\<Any,
Any\>(Any(), Any()) as Node\<K, V\> private val tail = Node\<Any,
Any\>(Any(), Any()) as Node\<K, V\> init { head.next = tail; tail.prev =
head } fun put(key: K, value: V, ttlMs: Long = Long.MAX_VALUE) { val
node = map.getOrPut(key) { Node(key, value) } node.value = value
node.expireAt = if (ttlMs == Long.MAX_VALUE) Long.MAX_VALUE else
System.currentTimeMillis() + ttlMs moveToFront(node) if (map.size \>
capacity) evict() } fun get(key: K): V? { val node = map\[key\] ?:
return null if (isExpired(node)) { remove(node); return null }
moveToFront(node) return node.value } private fun evict() { val expired
= map.values.firstOrNull { isExpired(it) } remove(expired ?:
tail.prev!!) } private fun isExpired(n: Node\<K, V\>) =
System.currentTimeMillis() \> n.expireAt private fun remove(n: Node\<K,
V\>) { n.prev?.next = n.next; n.next?.prev = n.prev; map.remove(n.key) }
private fun moveToFront(n: Node\<K, V\>) { remove(n); n.next =
head.next; n.prev = head head.next?.prev = n; head.next = n;
map\[n.key\] = n } }
