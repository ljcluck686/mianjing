## 1. 平时有使用过 AI 吗？AI 写代码怎么写比较高效？

**回答思路：** 展示真实使用经验 + 方法论

> "我日常高频使用 Cursor / GitHub Copilot，主要在以下场景提效：
>
> - **需求拆解阶段**：让 AI 帮我把需求拆成技术子任务，给出技术方案对比
> - **写代码阶段**：用好上下文，把相关文件、接口定义、数据结构一起喂给 AI，而不是孤立提问
> - **Code Review 阶段**：让 AI 扮演 Reviewer 找潜在 bug 和边界问题
> - **测试用例**：描述函数行为让 AI 生成边界测试
>
> 核心原则是：**我来设计，AI 来实现**。我不会让 AI 做架构决策，但可以让它帮我快速填充实现细节。"

## 2. AI 是怎么产生幻觉的？如何避免？

### 原理层面

| 原因                | 解释                                            |
| ------------------- | ----------------------------------------------- |
| 训练数据噪声        | 模型学到了错误的关联模式                        |
| 自回归生成机制      | 每个 token 基于概率预测，没有"不知道"的停止机制 |
| 知识截止 & 知识盲区 | 对训练集外的知识强行补全                        |
| 过度自信            | RLHF 训练使模型倾向于给出流畅自信的回答         |

### 避免方法

- **工程侧**：RAG 挂载外部知识库、引用溯源、输出结构化 JSON 便于校验
- **Prompt 侧**：加"如果不确定请说不知道"、要求引用来源、思维链（CoT）
- **系统侧**：多模型交叉验证、事实校验模块

## 3. 如何优化提示词？

### 分层回答法

```
基础层：角色设定 + 任务描述 + 输出格式
  "你是一个资深前端工程师，请帮我...，以 JSON 格式输出"

中级层：Few-shot 示例
  给 2-3 个输入输出例子，让模型对齐期望格式

高级层：Chain of Thought
  "请一步步思考，先分析...再给出..."

工程层：
  - 系统 Prompt 与用户 Prompt 分离
  - 动态注入上下文（RAG 召回结果）
  - Temperature 控制（创意任务调高，结构化输出调低至 0）
```

## 4. 外挂知识库 RAG 的原理

```
用户提问
    ↓
Embedding 模型 → 向量化问题
    ↓
向量数据库检索（cosine similarity 召回 Top-K 文档块）
    ↓
Prompt 拼装：[System] + [召回文档] + [用户问题]
    ↓
LLM 生成答案（基于真实文档，减少幻觉）
```

### 关键细节

- 文档切分策略（chunk size、overlap）
- 召回增强：混合检索（向量 + BM25 关键词）
- Rerank 模型二次排序提升精度
- 上下文窗口限制下的压缩策略

## 5. 工程和算法怎么平衡？（大模型应用工程师 vs 算法工程师）

> "我理解这个问题的本质是：**在大模型应用开发中，什么时候应该调模型，什么时候应该优化工程？**
>
> 我的判断框架是：
>
> - **先工程后算法**：Prompt 工程、RAG、工具调用这些工程手段成本低、迭代快，优先穷尽
> - **数据决定上限**：如果是领域特定任务，Fine-tune 前先看数据质量和规模是否达标
> - **成本收益比**：Fine-tune 一次模型的成本 vs 工程优化的迭代成本，商业项目里工程优先
> - **角色协作**：应用工程师负责链路稳定性、延迟、工具调用；算法工程师负责模型能力边界和评估体系
>
> 在飞书这种 To B 产品里，我认为工程可靠性权重更高，稳定输出比极致效果更重要。"

## 6. 手撕：二叉树最大宽度（LeetCode 662）

### 题意

每一层的宽度 = 该层最右节点位置 - 最左节点位置 + 1（中间 null 节点也算）

### 核心思路：BFS + 节点编号

```python
from collections import deque

def widthOfBinaryTree(root) -> int:
    if not root:
        return 0
    
    # 队列存 (节点, 编号)
    # 父节点编号 i → 左子 2*i，右子 2*i+1
    queue = deque([(root, 1)])
    max_width = 0
    
    while queue:
        level_size = len(queue)
        _, first_index = queue[0]  # 当前层第一个节点的编号
        
        for _ in range(level_size):
            node, index = queue.popleft()
            # 归一化编号，防止编号溢出（关键优化！）
            index -= first_index
            
            if node.left:
                queue.append((node.left, 2 * index))
            if node.right:
                queue.append((node.right, 2 * index + 1))
        
        # 最后一个出队的 index 就是当层最大编号
        max_width = max(max_width, index + 1)
    
    return max_width
```

### 面试加分点

| 亮点               | 说明                                                         |
| ------------------ | ------------------------------------------------------------ |
| **归一化防溢出**   | `index -= first_index` 每层重置，避免深度大时 2^n 溢出       |
| **时间复杂度**     | O(N)，每个节点访问一次                                       |
| **空间复杂度**     | O(W)，W 为最大宽度，最坏 O(N)                                |
| **为什么不用 DFS** | DFS 需要记录每层首尾节点，逻辑更复杂，BFS 天然按层遍历更直观 |
